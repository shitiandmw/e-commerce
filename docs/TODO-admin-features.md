# 后台管理待开发功能清单

> 参考网站：https://www.timecigar.com  
> 创建时间：2026-02-09  
> 状态标记：⬜ 未开始 | 🟡 进行中 | ✅ 已完成

---

## P0 - 核心功能（没有就无法运营）

### 1. ⬜ 文章/博客管理

**背景**：timecigar.com 有 495+ 篇文章，是网站内容运营的核心。

**需要实现**：
- 文章 CRUD（创建、编辑、删除、列表）
- 文章分类管理（雪茄快讯、品味生活、茄时在线等）
- 文章字段：标题、封面图、分类、摘要、正文（富文本）、发布日期、状态（草稿/已发布）
- 文章列表支持搜索、按分类筛选、分页
- 正文富文本支持嵌入产品链接
- 文章排序（置顶/手动排序）

### 2. ⬜ 首页 Banner / 轮播图管理

**背景**：首页有至少 4 组不同位置的轮播/Banner 区域，需要后台灵活配置。

**需要实现**：
- Banner 位置管理（主轮播、品牌聚焦、限量版故事、会员计划等）
- 每个位置下的 Banner 项 CRUD
- Banner 字段：图片、标题、副标题/描述、跳转链接、排序权重、启用/禁用
- 支持设置生效时间段（开始时间、结束时间）
- 图片可从媒体库选取

### 3. ⬜ 商品集合 / 策展列表管理

**背景**：首页有多个策展商品区域（炙热单品、古巴雪茄焦点推介、特选推荐等），需要后台手动选品。

**需要实现**：
- 商品集合 CRUD（名称、标识 key、描述）
- 集合内商品的添加/移除/排序
- 支持从商品列表中搜索并添加商品到集合
- 预定义集合位置（关联到前端展示区域）
- 支持 Tab 分组（如"特选推荐"下分：组合套装 / 限量版 / 特惠专区）

### 4. ⬜ 导航菜单管理

**背景**：网站有复杂的 Mega Menu，一级分类下有子分类和品牌列表，需要后台可视化编辑。

**需要实现**：
- 树形菜单编辑器（支持拖拽排序）
- 菜单项字段：名称、链接、图标/图片、父级、排序
- 支持多级嵌套（一级分类 > 二级标签 > 三级品牌列表）
- 品牌列表可关联已有品牌数据（自动拉取品牌 logo 和名称）
- 菜单项启用/禁用
- 支持多套菜单（主导航、Footer 导航等）

## P2 - 运营增强功能

### 5. ⬜ 静态页面管理

**背景**：网站 Footer 链接了多个静态内容页面（关于我们、服务条款、隐私政策、FAQ 等），需要后台编辑。

**需要实现**：
- 页面 CRUD（标题、URL slug、正文富文本、状态）
- 预置常用页面模板（关于我们、服务条款、隐私政策、FAQ、订单与发货、退换政策、品质保证等）
- 富文本编辑器（复用现有 RichTextEditor 组件）
- 页面列表与搜索

### 6. ⬜ 多语言内容管理

**背景**：网站支持繁中/EN/简中三种语言，产品名称有中英文双语显示（如 "Montecristo 蒙特"）。

**需要实现**：
- 产品表单增加多语言字段（标题、副标题、描述的 繁中/EN/简中 版本）
- 文章表单增加多语言字段
- 静态页面增加多语言字段
- 语言版本切换 UI（编辑时可切换语言 Tab 填写不同语言内容）
- 多语言字段的存储方案设计（metadata / 独立字段 / 关联表）

### 7. ⬜ 公告栏管理

**背景**：首页顶部有滚动公告条，轮播显示消息（如"迎新礼遇"、"尊享全球免运费"等）。

**需要实现**：
- 公告项 CRUD（文案、跳转链接、排序、启用/禁用）
- 支持设置生效时间段
- 公告列表排序
- 前端轮播顺序与后台排序一致

### 8. ⬜ 弹窗管理

**背景**：网站有营销弹窗（如注册引导弹窗），需要后台配置。

**需要实现**：
- 弹窗 CRUD
- 弹窗字段：图片、标题、描述、按钮文案、按钮链接、启用/禁用
- 触发条件配置（首次访问 / 每次访问 / 指定页面）
- 显示频率控制（每天一次 / 每会话一次 / 仅一次）

### 9. ⬜ 产品标签/徽章管理

**背景**：商品卡片上有标签如"珍藏版"、包装规格"5支/盒"等。

**需要实现**：
- 标签 CRUD（名称、颜色/样式、类型）
- 标签类型区分（徽章类如"珍藏版"、属性类如"5支/盒"）
- 产品表单中增加标签关联（可多选）
- 标签列表管理

### 10. ⬜ SEO 管理

**背景**：产品、文章、页面都需要 SEO 字段以优化搜索引擎表现。

**需要实现**：
- 产品表单增加 SEO 区块（meta title、meta description、og:image）
- 文章表单增加 SEO 区块
- 静态页面表单增加 SEO 区块
- SEO 字段预览（模拟搜索引擎展示效果）
- 支持自动从标题/描述生成默认 SEO 内容

## 实施建议

**推荐开发顺序**：
1. 文章管理 → 2. Banner 管理 → 3. 商品集合 → 4. 导航菜单 → 5. 静态页面 → 6. SEO → 7. 产品标签 → 8. 公告栏 → 9. 弹窗 → 10. 多语言内容

**说明**：先做 P0 四个核心模块，确保前端页面能跑通。P2 模块可穿插开发，其中"多语言内容"涉及数据结构改动较大，建议放到最后整体规划。
